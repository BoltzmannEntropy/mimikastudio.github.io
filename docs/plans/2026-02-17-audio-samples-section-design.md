# Audio Samples Section Design

## Overview
Add an audio samples section to the MimikaStudio website where users can listen to TTS demos generated by Mimika Studio, organized by model.

## Requirements
- Display samples for three TTS engines: Kokoro, Qwen3-TTS, ChatterBox
- For voice cloning demos (Qwen3, ChatterBox): show reference voice alongside generated output
- Place section after the "3 Engines" section
- Match existing website styling (cards, borders, hover effects)

## Available Samples

### Kokoro (Built-in Voices)
- emma, george, lily, daniel, isabella, fable
- Source: `/MimikaCODE/backend/data/samples/kokoro/`

### Qwen3-TTS (Voice Cloning)
- natasha-genesis4, suzan-genesis4, aiden-genesis4, ryan-genesis4, natasha-hebrew
- Source: `/MimikaCODE/backend/data/pregenerated/`

### ChatterBox (Emotional Speech)
- natasha-demo, suzan-demo
- Source: `/MimikaCODE/backend/data/pregenerated/`

### Reference Voices
- Natasha.wav, Suzan.wav, Max.wav, Sara.wav
- Source: `/MimikaCODE/backend/data/samples/voices/`

## File Structure
```
MimikaWEB/
├── audio/

│   ├── qwen3/

│   └── references/

```

## UI Design

### Section Header
- Label: "AUDIO SAMPLES"
- Title: "Hear Mimika in action"
- Subtitle describing the demo section

### Model Subsections
Each model gets a subsection with:
- Model name and brief description
- Grid of sample cards

### Card Types
1. **Simple card** (Kokoro): Voice name, description, single play button
2. **Comparison card** (Qwen3, ChatterBox): Voice name, reference player + generated player side by side

### Audio Player
- Circular play/pause button with accent color
- Progress bar when playing
- Native HTML5 `<audio>` element (no external libraries)

## Technical Approach
- Copy audio files to website's `/audio/` folder
- Add CSS for new card styles within existing `<style>` block
- Add JavaScript for audio playback at end of file
- Responsive design: cards stack on mobile
